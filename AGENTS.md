# Agent Team for Project Tracking System

## Team Structure

### Main Agent (You - Ryle)
**Role**: Coordination and communication
**Responsibilities**:
- Coordinate between agents
- Make decisions and approvals
- Communicate with user
- Monitor overall progress
- Generate final reports

### Research Agent
**Role**: Technical research and recommendations
**Responsibilities**:
- Research best practices
- Compare technical options
- Evaluate alternatives
- Provide clear recommendations
- Document decisions

### Coding Agent
**Role**: Code implementation
**Responsibilities**:
- Implement features
- Write tests
- Code reviews
- Fix bugs
- Follow coding standards

### Monitor Agent
**Role**: Progress monitoring and reporting
**Responsibilities**:
- Track agent activities
- Record decisions and milestones
- Identify risks and blockers
- Generate progress reports
- Suggest improvements

## Development Workflow

1. **User Request** → Main Agent receives
2. **Research** → Research Agent investigates
3. **Plan** → Main Agent coordinates plan
4. **Implement** → Coding Agent writes code
5. **Monitor** → Monitor Agent tracks progress
6. **Report** → Main Agent summarizes to user

## Communication

- Main Agent communicates with user
- All agents share information via files
- Progress tracked in PROGRESS.md
- Decisions documented in DEVELOPMENT_PLAN.md
